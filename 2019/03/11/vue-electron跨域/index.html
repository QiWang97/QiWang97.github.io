在做vue-electron 项目开发时，突然发现了一个‘巧妙的’伪造请求头的方法。

vue-electron 伪造请求头

简介

  • 使用 jsonp 接口时，referer 属性可以提供接口域名进行限制访问。
  • web 安全限制，浏览器不允许发送请求时直接设置 referer 和 host

示例

QQ音乐 歌词数据接口：

GET：https://c.y.qq.com/lyric/fcgi-bin/fcg_query_lyric_new.fcg

参数：songmid 歌曲 mid，示例使用 004aHCfZ3VDYG3

请求头： "Host": "c.y.qq.com"，

​ "Referer": "https://y.qq.com/portal/player.html"

electron 同源策略

electron 可以设置使用启用同源策略

webSecurity设置 web 安全性，如果参数设置为 false，它将禁用同源策略

allowDisplayingInsecureContent表示是否允许一个使用 https 的界面来展示由 http URLs 传过来的资源。默认false。
allowRunningInsecureContent表示是否允许一个使用 https 的界面来渲染由 http URLs 提交的 html，css，javascript。默认为 false。

electron 后台运行程序

Node.js 中的child_process，可以创建子进程，在子进程执行操作

而同时，electron 也支持该操作

electron 通信

electron 使用 ipcMain 与 ipcRenderer 进行主线程与渲染线程的通信。这里不再赘述。

问题与解决思路

以上操作在 vue-electron 中对接口请求头伪造失败，vue 提示错误

当看到 child_process 时，我发现子进程完全可以后台运行 js 脚本执行 axios.get，跳过 vue与浏览器限制。那么，问题就转化成如何传入参数并获得子进程数据。

解决方法为：

  • electron 执行 exec 命令行启动后台脚本，并传入参数
  • js 脚本对参数解析，发送请求，输出结果
  • 子进程监听程序的正常输出（即请求结果），发送数据到渲染进程。

3. api 文件

lyric.js 使用 commander监听命令行传入的参数，使用 axios.get 进行跨域伪造请求

注： 该lyric.js可以直接使用 node lyric.js --message ‘004aHCfZ3VDYG3’获取结果

备注

警告：该方法在执行命令行时可以进行参数解析和判断，但不排除 shell 脚本攻击。因此，使用需谨慎。
